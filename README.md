# Colour-Science-XYZ

Python code associated with the methods developed in **"Automating Diffuse Reflectance Spectroscopy for High-Throughput Colorimetric Characterization "** by Ronald Ceballos, Juan Brito, Leonardo Velasco.

> **Contact:** rceballosl@unal.edu.co · jbritoo@unal.edu.co · lvelascod@unal.edu.co

---

## Overview

This repository contains a Jupyter notebook that implements a complete colorimetry pipeline for reflectance spectroscopy data, following the standards of the International Commission on Illumination (CIE):

- **Observer:** CIE 1931 2-Degree Standard Observer
- **Illuminant:** D65 (standard daylight)

The workflow takes raw spectral reflectance data (as CSV files exported by a spectrophotometer) and produces:

| Output | Description |
|--------|-------------|
| Tristimulus XYZ | CIE 1931 colorimetric values |
| CIELab (L\*, a\*, b\*) | Perceptually uniform colour-space coordinates |
| sRGB | Display-ready colour values with gamut mapping |
| Delta E 00 ($\Delta E^*_{00}$) | Perceptual color difference against a reference sample |
| Reflectance spectra plot | Overlay of all sample curves |
| CIE 1931 xy chromaticity diagram | Chromaticity locus with sample positions |
| CIELAB channel maps | Spatial L\*, a\*, b\* decomposition |
| sRGB colour charts | Visual colour swatches (with and without labels) |

## Gradio Web App

In addition to the notebook, this repository includes a Gradio-based graphical interface (`app.py`) that exposes the same colorimetry pipeline through an interactive, browser-based UI. It is intended for users who prefer a point-and-click workflow over editing notebook cells.

### Launching the app

```bash
conda activate colorimetry
python app.py
```

Gradio will start a local web server and print a URL (typically `http://127.0.0.1:7860`) that you can open in your browser.

### App Tabs

| Tab | Purpose |
|-----|---------|
| 1. Data Loading | Upload a Raw spectrophotometer CSV (reshaped and saved automatically) or an already Processed CSV (used as-is, not re-saved). |
| 2. Reflectance Spectra | Plots the reflectance curves for a chosen wavelength range and exports the figure as PNG. |
| 3. Color Space Data (MSDS, XYZ, LAB, sRGB) | Computes and previews the aligned MSDS spectra plus the XYZ, CIELAB and sRGB tristimulus tables, exporting each as a separate Excel file. |
| 4. Color Spaces (XYZ, Lab, sRGB) | Generates the CIE 1931 chromaticity diagram, the CIELAB channel decomposition, and the sRGB sample grid, based on a user-defined rows × columns matrix layout. |
| 5. Delta E00 Analysis | Computes $\Delta E^*_{00}$ against a configurable, expandable table of reference colors (HEX or [L, a, b]), grouped by a user-defined sample group size. |

### Design Notes

- If a Raw file is uploaded, it is reshaped and saved to `data-processed/` exactly as in Section 2 of the notebook. If a Processed file is uploaded, it is used directly from its original location and is *not* written again to `data-processed/`.
- The reference-colors table in the Delta E tab supports adding or removing rows freely, so any number of reference colors can be defined (not limited to 6).

## XYZ Stage (G-code & CAD)
 
The `xyz/` folder contains everything related to the physical XYZ motion stage used to automate sample measurement:
 
- **`XYZ_Stage_Gcode_Generator.ipynb`** — generates a G-code program that sweeps the stage across a grid of points, activating a sensor at each position (see the notebook itself for usage details).
- **`assembly/`** — the full stage assembly, provided in STEP format (CAD-agnostic) and as a native Fusion 360 file.
- **`parts/`** — the individual components of the stage, each as a separate STEP file.
This folder is independent from the colorimetry pipeline (notebook + Gradio app) and only needs to be consulted if you are building, modifying, or manufacturing the stage itself.

## Input Data Format

The notebook expects raw CSV files in the format produced by many commercial spectrophotometers, where each sample occupies two columns (Wavelength | %R), all columns placed side by side:

```
POS0             | (unnamed) | POS1             | (unnamed) | …
Wavelength (nm)  | %R        | Wavelength (nm)  | %R        | …
400              | 12.5      | 400              | 13.1      | …
401              | 12.8      | 401              | 13.4      | …
…
```

> **Section 2** of the notebook reshapes this format into a single tidy DataFrame and saves it to `data-processed/`.  
> If your data is already clean (one wavelength column + one %R column per sample), you can skip Section 2 and load the file directly in **Section 4**.

### Baseline 100% T column

Some spectrophotometers prepend a **Baseline 100% T** column to every exported CSV. This column records the instrument's own reference measurement and is not a real sample. After processing, it becomes `Reflectance POS 0` in the tidy DataFrame.

**What to do:**

- If your instrument writes a baseline column, set `DROP_BASELINE = True` in the **Section 2** user-parameters block. The cell will drop `Reflectance POS 0` automatically before saving the processed file.
- If your instrument does not write a baseline column, leave `DROP_BASELINE = False` (the default) and the first column will be treated as a normal sample.

## Colorimetry Background

### CIE 1931 2-Degree Standard Observer

The CIE 1931 colour-matching functions (CMFs) $\bar{x}(\lambda)$, $\bar{y}(\lambda)$, $\bar{z}(\lambda)$ describe how a human observer with a 2° field of view matches colours. They form the foundation of all modern colorimetry.

### D65 Illuminant

D65 represents average daylight (correlated colour temperature ≈ 6504 K). It is the standard illuminant for most colour-science applications and sRGB displays.

### XYZ → CIELab

$$L^* = 116 f\!\left(\frac{Y}{Y_n}\right) - 16, \quad
a^* = 500\left[f\!\left(\frac{X}{X_n}\right) - f\!\left(\frac{Y}{Y_n}\right)\right], \quad
b^* = 200\left[f\!\left(\frac{Y}{Y_n}\right) - f\!\left(\frac{Z}{Z_n}\right)\right]$$

where $f(t) = t^{1/3}$ for $t > (6/29)^3$ and a linear approximation otherwise, and $(X_n, Y_n, Z_n)$ are the D65 white-point tristimulus values.

### Gamut Mapping

When converting XYZ to sRGB, some colours may fall outside the sRGB gamut (channel values > 1 or < 0). The notebook applies a proportional scaling that clips negative values to zero and scales all channels by the maximum, preserving hue while reducing saturation.
